# Newt CMS セットアップガイド

このガイドに従って、Newt CMSの設定を行ってください。

## 1. Newtアカウント＆スペース作成

1. https://app.newt.so/ にアクセス
2. 新しいスペースを作成
   - スペース名: `sportscommission`（任意）
   - Space UID: 後で使用するのでメモ

## 2. Appの作成

1. 左サイドバーから「App」を作成
2. App名: `website`
3. App UID: `website`（後で使用）

## 3. モデルの定義

### 📝 Newsモデル（ニュース記事）

以下のフィールドを持つ「News」モデルを作成してください。

| フィールド名 | フィールドID | タイプ | 必須 | 説明 |
|------------|------------|------|-----|-----|
| タイトル | `title` | テキスト | ✓ | 記事タイトル |
| スラッグ | `slug` | テキスト | ✓ | URLパス（例: `news-2024-01-01`） |
| 公開日 | `publishedAt` | 日時 | ✓ | 記事の公開日時 |
| カバー画像 | `coverImage` | 画像 | - | アイキャッチ画像 |
| カテゴリー | `category` | セレクト | - | 選択肢: お知らせ/イベント/レポート/その他 |
| 本文 | `body` | リッチテキスト | ✓ | 記事本文（HTML対応） |
| メタ説明 | `metaDescription` | テキスト | - | SEO用のディスクリプション（160字以内推奨） |

### 📄 Pageモデル（固定ページ）

「About」「Contact」などの固定ページ用モデル。

| フィールド名 | フィールドID | タイプ | 必須 | 説明 |
|------------|------------|------|-----|-----|
| タイトル | `title` | テキスト | ✓ | ページタイトル |
| スラッグ | `slug` | テキスト | ✓ | URLパス（例: `about`） |
| 本文 | `body` | リッチテキスト | ✓ | ページ本文 |
| メタ説明 | `metaDescription` | テキスト | - | SEO用 |

## 4. APIキーの取得

1. 左サイドバーから「Settings」→「API keys」
2. 「CDN API token」をコピー（公開用の読み取り専用トークン）
3. メモしておく

## 5. 必要な情報まとめ

以下の情報を `.env.local` に設定します：

```bash
NEWT_SPACE_UID=your-space-uid
NEWT_APP_UID=website
NEWT_API_TOKEN=your-cdn-api-token
```

## 6. テストコンテンツの作成

動作確認のため、以下のテストデータを作成してください：

### Newsの例
- **タイトル**: 「日本スポーツコミッション公式サイトをリニューアルしました」
- **スラッグ**: `site-renewal-2026`
- **公開日**: 今日の日付
- **カテゴリー**: お知らせ
- **本文**: 適当な文章（「この度、公式サイトをリニューアルしました...」など）

### Pageの例（About）
- **タイトル**: 「私たちについて」
- **スラッグ**: `about`
- **本文**: 団体の説明文
